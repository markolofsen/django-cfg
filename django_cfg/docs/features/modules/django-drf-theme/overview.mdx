# DRF Theme

Django-CFG's **DRF Theme** replaces the default Bootstrap-based DRF Browsable API with a modern Tailwind CSS v4 interface. It extends `BrowsableAPIRenderer` so all existing DRF functionality is preserved.

## Features

- **Dark / Light / Auto mode** — three-mode theme system persisted in a cookie; auto-mode follows system preference
- **Glass morphism UI** — translucent design with backdrop blur
- **Syntax-highlighted JSON** — Prism.js powered viewer with collapsible nodes
- **One-click copy** — copy JSON responses and endpoint URLs
- **Keyboard shortcuts** — full keyboard navigation including command palette (`⌘K`)
- **Responsive layout** — mobile-first, works on all screen sizes
- **Alpine.js** — lightweight reactivity, no jQuery dependency
- **88% smaller bundle** — 33 KB total vs 278 KB for the old Bootstrap 3 theme

## Quick Start

Add `TailwindBrowsableAPIRenderer` to your DRF renderer classes. Django-CFG does this automatically when `enable_drf_tailwind=True` (the default):

```python
# Automatic (default in DjangoConfig)
class MyConfig(DjangoConfig):
    enable_drf_tailwind = True  # default — no extra configuration needed
```

To configure DRF renderer classes explicitly:

```python
from django_cfg.models.api.drf import DRFConfig

class MyConfig(DjangoConfig):
    drf = DRFConfig(
        renderer_classes=[
            "rest_framework.renderers.JSONRenderer",
            "django_cfg.modules.django_drf_theme.TailwindBrowsableAPIRenderer",
        ]
    )
```

To switch back to the default Bootstrap theme:

```python
class MyConfig(DjangoConfig):
    enable_drf_tailwind = False
```

## TailwindBrowsableAPIRenderer

`TailwindBrowsableAPIRenderer` extends `BrowsableAPIRenderer` from Django REST Framework.

**Template:** `rest_framework/tailwind/api.html`

**Additional context variables:**

| Variable | Values | Description |
| --- | --- | --- |
| `theme` | `"light"`, `"dark"`, `"auto"` | Theme from request cookie `theme` |
| `html_class` | `""` or `"dark"` | CSS class applied to `<html>` |
| `tailwind_version` | `4` | Tailwind CSS version in use |
| `enable_dark_mode` | `True` | Dark mode enabled flag |
| `enable_shortcuts` | `True` | Keyboard shortcuts enabled flag |
| `enable_animations` | `True` | CSS animations enabled flag |

Theme is read from the `theme` cookie. If not set, `"auto"` is used (system preference via CSS media query).

## Template Structure

```
templates/rest_framework/tailwind/
├── base.html                    # Navbar, theme switcher, shortcuts handler
├── api.html                     # Main API content, JSON viewer, request forms
└── forms/
    ├── raw_data_form.html       # POST/PUT/PATCH body form
    └── filter_form.html         # Query parameter filter sidebar
```

Override any template by placing a file at the same relative path inside your app's `templates/` directory.

## Template Tags

Load via `{% load tailwind_tags %}` or `{% load tailwind_info %}`.

### `tailwind_tags` Library

| Tag / Filter | Description |
| --- | --- |
| `{% tailwind_css %}` | Renders `<script>` tags for Tailwind CDN v4 with `forms`, `typography`, `aspect-ratio`, and `container-queries` plugins; sets `darkMode: 'class'` |
| `\|add_class:"classes"` | Adds CSS classes to a Django form field widget |
| `\|get_item:key` | Safe dictionary key access in templates |
| `{% get_theme_class theme %}` | Returns `"dark"` or `""` based on theme string |
| `\|prettify_json` | Formats a JSON string or object with 2-space indentation |

### Example

```html
{% load tailwind_tags %}
<!DOCTYPE html>
<html class="{% get_theme_class theme %}">
<head>
  {% tailwind_css %}
</head>
<body>
  <form>
    {{ form.body|add_class:"border rounded px-3 py-2 w-full" }}
  </form>
</body>
</html>
```

## Theme Detection

The theme is resolved in this order:

1. `theme` cookie (set by the theme toggle button)
2. `auto` — uses `prefers-color-scheme` CSS media query to follow system preference
3. Default: `auto`

Selecting light or dark in the interface sets the `theme` cookie to `"light"` or `"dark"` so the preference persists across page loads.
