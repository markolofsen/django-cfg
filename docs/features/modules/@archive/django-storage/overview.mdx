# Storage Configuration

<Callout type="info">
`django-storage` is not a standalone module. `StorageConfig` is a Pydantic configuration model in `django_cfg.models.django.storage` that configures Django's storage backends. See `django-cleanup` for file cleanup functionality.
</Callout>

## StorageConfig

`StorageConfig` is a Pydantic v2 model set on `DjangoConfig.storage`. It configures how Django-CFG's file cleanup module handles `FileField` and `ImageField` deletion.

```python
from django_cfg import DjangoConfig, StorageConfig

class MyConfig(DjangoConfig):
    storage = StorageConfig(
        enabled=True,
        auto_cleanup=True,
        delete_on_replace=True,
        log_deletions=False,
        exclude_models=set(),
        exclude_fields=set(),
        check_shared_files=False,
        respect_soft_delete=True,
        soft_delete_fields={"deleted_at", "is_deleted", "deleted"},
    )
```

### Fields

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `enabled` | `bool` | `True` | Enable the file cleanup module entirely |
| `auto_cleanup` | `bool` | `True` | Register signals for all models with file fields at startup |
| `delete_on_replace` | `bool` | `True` | Delete old file when a field is updated with a new value |
| `exclude_models` | `set[str]` | `set()` | Models to skip entirely. Format: `"app_label.ModelName"` |
| `exclude_fields` | `set[str]` | `set()` | Fields to skip. Format: `"app_label.ModelName.field_name"` |
| `log_deletions` | `bool` | `False` | Log each deleted file at `INFO` level |
| `check_shared_files` | `bool` | `False` | Check if another record references the same file before deleting |
| `respect_soft_delete` | `bool` | `True` | Skip file deletion for soft-deleted instances |
| `soft_delete_fields` | `set[str]` | `{"deleted_at", "is_deleted", "deleted"}` | Field names that indicate a soft-delete pattern |

`exclude_models` and `exclude_fields` accept lists — they are automatically converted to sets.

## Import Paths

```python
# Primary (recommended)
from django_cfg import StorageConfig

# Explicit paths
from django_cfg.models import StorageConfig
from django_cfg.models.django.storage import StorageConfig
```

## Django Storage Backends

Django-CFG does not manage which storage backend Django uses — that is configured via standard Django settings. To use S3, GCS, or another backend, configure Django's `STORAGES` setting directly:

```python
# In your DjangoConfig or settings.py

# S3 via django-storages
STORAGES = {
    "default": {
        "BACKEND": "storages.backends.s3boto3.S3Boto3Storage",
        "OPTIONS": {
            "bucket_name": "my-bucket",
            "region_name": "us-east-1",
        },
    },
    "staticfiles": {
        "BACKEND": "django.contrib.staticfiles.storage.StaticFilesStorage",
    },
}
```

The File Cleanup module works with any storage backend that implements Django's standard `Storage` interface, including local filesystem, S3, GCS, and Azure Blob storage.

## Fallback: `settings.DJANGO_STORAGE`

If `DjangoConfig` is not used, configure cleanup via Django settings:

```python
# settings.py
DJANGO_STORAGE = {
    "auto_cleanup": True,
    "delete_on_replace": True,
    "log_deletions": False,
    "exclude_models": ["backups.DatabaseBackup"],
    "exclude_fields": ["documents.Contract.original_scan"],
    "check_shared_files": False,
    "respect_soft_delete": True,
    "soft_delete_fields": ["deleted_at", "is_deleted", "deleted"],
}
```
