# Monitor

Full-stack error tracking and performance monitoring built into Django-CFG. Captures browser events and server-side errors into a unified admin dashboard — zero configuration required.

<Callout type="info">
Monitor is a built-in app. It is always available once `django_cfg.apps.system.monitor` is in `INSTALLED_APPS` (registered automatically by Django-CFG).
</Callout>

## What Gets Tracked

| Capture path | What is tracked |
|---|---|
| Browser SDK (`@djangocfg/monitor`) | JS errors, network failures, console warnings, Zod validation errors, page events |
| `got_request_exception` signal | Unhandled HTTP 500 errors |
| `logging.Handler` | ERROR and CRITICAL log records |
| `execute_wrapper` | Slow database queries |
| RQ `ExceptionHandler` | Failed background jobs |
| `MonitoringWorker` | OOM / SIGKILL process kills |
| `capture` API | Manual exception and message capture |

All server-side capture paths are **fire-and-forget** — they never raise and never break the calling code.

## Quick Start

### Backend

The monitor app is enabled automatically. For manual captures from your application code:

```python
from django_cfg.apps.system.monitor.capture import capture

try:
    process_payment(order)
except Exception:
    capture.exception(extra={"order_id": order.id})

capture.message("Suspicious login attempt", level="warning", extra={"ip": ip})
```

### Frontend (React / Next.js)

Install the SDK:

```bash
pnpm add @djangocfg/monitor
```

Add the built-in provider to your root layout — no extra component needed:

```tsx
// app/layout.tsx
import { MonitorProvider } from '@djangocfg/monitor/client'

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>
        <MonitorProvider project="my-app" environment={process.env.NODE_ENV} />
        {children}
      </body>
    </html>
  )
}
```

Once initialized, JS errors, unhandled rejections, and console warnings are captured automatically.

---

## Admin Dashboard

Three views are available under the **Monitor** section in Django admin:

- **Events** — `FrontendEvent` list with badge-colored types, search, filters
- **Sessions** — `AnonymousSession` list showing visitor → user linking status
- **Server Events** — triage dashboard with resolve/reopen bulk actions
